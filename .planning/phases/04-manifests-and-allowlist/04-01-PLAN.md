<objective>
Two additions:

1. **Tool allowlist (HARN-07)**: each harness class declares `TOOL_WHITELIST: frozenset[str]`. The base class asserts at every `_step_model` call that the tools payload set matches the harness's whitelist. Violation raises `ToolAllowlistViolation`.

2. **Run manifests (RUN-04)**: the runner writes `runs_expected.jsonl` before starting, appends to `runs_completed.jsonl` after each cell. A `--resume <run_id>` path executes only the missing (harness, task, seed) triples.

**Freeze tag move**: this phase edits gated files (harnesses/). Because no matrix has been run against the old tag, re-tagging `harnesses-frozen` at the new HEAD after this phase is methodologically clean — no measurements are invalidated. HARNESSES_FROZEN.md is updated to document the move and the new per-file blob SHAs.
</objective>
