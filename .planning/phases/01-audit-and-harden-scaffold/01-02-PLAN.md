    Required edit:
    ```python
    import re
    import unicodedata

    _ASCII_WS = re.compile(r"[ \t\n\r\f\v]+")  # NOT \s — \s is Unicode-aware, varies on NBSP

    def _norm(s: str) -> str:
        # 1. NFC: visually identical graphemes become byte-identical
        s = unicodedata.normalize("NFC", s)
        # 2. Strip leading/trailing whitespace (including Unicode — .strip() is OK here,
        #    we are removing, not comparing)
        s = s.strip()
        # 3. casefold (NOT .lower()) — locale-independent; handles German ß -> ss,
        #    Turkish dotted/dotless i, Greek sigma
        s = s.casefold()
        # 4. Collapse ASCII whitespace runs to a single space (explicit class, not \s)
        s = _ASCII_WS.sub(" ", s)
        return s
    ```

    Placement rules:
    - Put `_ASCII_WS` as a module-level compiled regex (not inside the function) — compilation cost matters on large matrix runs.
    - Keep the `import re` and `import unicodedata` at the top of the file with the other imports.
    - If `grader.py` currently coerces non-string inputs anywhere (e.g., `str(x)` for numeric fields), preserve that coercion — only change the normalization pipeline.
    - If there is an existing `_NORM_RE` or similar module-level regex for the old `\s+` pattern, remove it.

    Pitfalls to honor (from 01-RESEARCH.md and PITFALLS.md Pitfall 6):

    ```python
    """Golden-trace determinism: grading the same input 100x yields byte-identical output.

    Pins the hardened _norm pipeline (NFC -> strip -> casefold -> ASCII-ws-collapse)
    against Turkish-i, German-ß, NFC-vs-NFD composed graphemes, and whitespace edges.
    """
    from __future__ import annotations

    import json

    from harness_eng.grader import grade


    # (predicted, expected, expected_success) — each row pins one normalization invariant.
    GOLDEN_CASES = [
        # Case, whitespace, and basic equivalence
        ({"name": "Hello World"}, {"name": "hello world"}, True),
        ({"name": "  HELLO\tWORLD\n"}, {"name": "hello world"}, True),
        # German ß — casefold normalizes to 'ss'; .lower() would keep ß and mismatch
        ({"greeting": "straße"}, {"greeting": "strasse"}, True),
        # NFC vs NFD — 'é' as single codepoint vs 'e' + combining acute; NFC collapses both
        ({"word": "café"}, {"word": "café"}, True),
        # Wrong answer stays wrong
        ({"a": "foo"}, {"a": "bar"}, False),
    ]


    def test_golden_cases():
        for predicted, expected, expect_success in GOLDEN_CASES:
            r = grade(predicted, expected)
            assert r.success is expect_success, (
                f"case failed: predicted={predicted!r} expected={expected!r} "
                f"got success={r.success!r} want {expect_success!r}"
            )


    def test_100x_determinism():
        """Same input graded 100 times — all result dicts byte-identical when serialized."""
        predicted = {"name": "  Hello  WORLD  ", "price": "19.99"}
        expected = {"name": "hello world", "price": "19.99"}
        results = [grade(predicted, expected) for _ in range(100)]
        # Serialize each result to canonical JSON; set() reduces to unique serializations.
        serialized = {
            json.dumps(
                {
                    "per_field": r.per_field,
                    "field_accuracy": r.field_accuracy,
                    "success": r.success,
                },
                sort_keys=True,
            )
            for r in results
        }
        assert len(serialized) == 1, f"grader non-deterministic: {len(serialized)} distinct outputs"
    ```

    Adjustments:
    - The attribute names `per_field`, `field_accuracy`, `success` on the `GradeResult` (or equivalent) returned by `grade()` MUST match the current dataclass/type — inspect `grader.py` first and adjust accordingly. Do NOT change the grader surface.
    - If `GradeResult` uses different field names (e.g., `per_field_scores`, `overall_accuracy`, `passed`), update the test's `json.dumps(...)` accordingly. Keep the semantics (fields-with-per-field-scores + aggregate + boolean).
