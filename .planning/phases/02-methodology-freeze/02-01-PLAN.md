<objective>
Make the methodology tamper-evident. After this plan lands:
- The pre-registered hypothesis is committed to README before the freeze tag
- HARNESSES_FROZEN.md pins commit SHA + per-file SHAs of the gated set (harnesses/, tools.py, model.py)
- The git tag `harnesses-frozen` marks that commit
- The runner refuses to execute if any gated file has diverged from the tag
- A test exercises both the fail and pass paths of the gate

Note: This plan is written inline by the orchestrator rather than via gsd-planner + gsd-plan-checker agents because the phase is narrow (4 reqs, 4 success criteria, all specified concretely in ROADMAP.md) and project-level research (`.planning/research/`) already covers every relevant pitfall.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Append pre-registered hypothesis to README.md</name>
  <files>README.md</files>
  <action>
    Add a `## Pre-registered hypothesis` section near the top of README.md (just after "The setup"). The paragraph must name:
    - Expected direction (which harnesses will win on success rate, which will win on cost-per-success)
    - Rough magnitude (2–4x spread in success rate, 5–10x spread in cost — per the article pitch)
    - Date of pre-registration (2026-04-23)
    - That any edit to this section after the freeze tag is a methodological defect, not a clarification

    Commit this section BEFORE creating the `harnesses-frozen` tag — git history will show the ordering, which is the whole point.
  </action>
</task>

<task type="auto">
  <name>Task 2: Add freeze-gate check to runner.py</name>
  <files>src/harness_eng/runner.py</files>
  <action>
    Add a `check_freeze_gate() -> None` function that:
    - Runs `git diff harnesses-frozen HEAD -- <gated paths>` via subprocess
    - Gated paths: `src/harness_eng/harnesses/`, `src/harness_eng/tools.py`, `src/harness_eng/model.py`
    - If the command exits nonzero (tag missing), raises FreezeGateError with message "harnesses-frozen tag not found — create with: git tag harnesses-frozen <sha>"
    - If the diff is non-empty, raises FreezeGateError naming the offending files
    - Accepts an env var `HARNESS_ENG_SKIP_FREEZE_GATE=1` to bypass (tests only)

    Call `check_freeze_gate()` at the top of `run_matrix()`. Define `FreezeGateError` in runner.py.
  </action>
</task>

<task type="auto">
  <name>Task 3: Write HARNESSES_FROZEN.md with per-file SHAs</name>
  <files>HARNESSES_FROZEN.md</files>
  <action>
    Before running the tag, gather:
    - `git rev-parse HEAD` (the freeze commit SHA)
    - For each gated file, `git rev-parse HEAD:<path>` (the blob SHA)
    - Date: 2026-04-23

    Write HARNESSES_FROZEN.md at repo root recording these. The file MUST be committed BEFORE the tag is created — the tag points at the commit that contains this file.
  </action>
</task>

<task type="auto">
  <name>Task 4: Tests for the freeze gate</name>
  <files>tests/test_freeze_gate.py</files>
  <action>
    Two tests, both via mocked subprocess (do not require a real git repo):
    1. `test_freeze_gate_raises_on_diverged_gated_file` — mock subprocess.run to return non-empty diff output for a gated path; assert FreezeGateError is raised; assert the offending path appears in the message
    2. `test_freeze_gate_passes_on_clean_diff` — mock subprocess.run to return empty diff; assert no exception
    3. `test_freeze_gate_raises_when_tag_missing` — mock subprocess.run to return nonzero exit; assert FreezeGateError with the "tag not found" message
  </action>
</task>

<task type="auto">
  <name>Task 5: Commit, then create the tag</name>
  <files>(git metadata)</files>
  <action>
    After tasks 1–4 land, commit the aggregate change (README + HARNESSES_FROZEN.md + runner.py + test_freeze_gate.py). THEN create the annotated tag at the resulting commit:
    `git tag -a harnesses-frozen -m "methodology freeze for harness_eng matrix run" <sha>`

    Verify:
    - `git tag -l harnesses-frozen` prints the tag
    - `git diff harnesses-frozen HEAD -- src/harness_eng/harnesses/ src/harness_eng/tools.py src/harness_eng/model.py` is empty
    - `pytest tests/test_freeze_gate.py -v` — all 3 tests pass
  </action>
</task>
